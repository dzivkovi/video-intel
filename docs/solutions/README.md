# Solved Problems

Documents recording how specific problems were diagnosed and resolved.

## Three-Bucket Rule

| Bucket | Examples | Rule |
|--------|----------|------|
| **Living docs** | CLAUDE.md, ARCHITECTURE.md | Always rewrite to current truth |
| **Historical docs** | work/ session notes, old plans | Never rewrite |
| **Decision records** | docs/adr/, docs/solutions/ | Append notes, never rewrite |

## Naming Convention

`{category}/{description}-{YYYYMMDD}.md`

Categories: `workflow-issues/`, `integration-issues/`, `process-issues/`
